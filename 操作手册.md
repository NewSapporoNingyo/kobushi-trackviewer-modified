## 1. 概述

Kobushi Trackviewer 是一个 Python 3 GUI 应用程序，用于读取 **BVE Trainsim 5/6** 的**地图文件（map file，.txt 格式）**，生成线路的：

- **平面图（Plan View）**：轨道在水平面（XY平面）上的投影
- **纵断面图（Profile View）**：里程-标高关系图（含坡度变化）
- **曲线半径图（Curve Radius Graph）**：里程-曲线半径关系图

### 1.1 主要功能

| 功能 | 说明 |
|------|------|
| 地图文件解析 | 基于 Lark 语法解析器，支持 BVE Map 2.00+ 格式的全部语法（数学表达式、变量、`rand()`、`$include` 指令） |
| 轨道坐标计算 | 在每个控制点上生成完整的 3D 坐标（x, y, z），包含方位角、曲率半径、坡度、超高（Cant）等 |
| 平面图 | 轨道 XY 平面投影，支持旋转、平移、缩放，带比例尺和坐标网格 |
| 纵断面图 | 里程-标高图，显示坡度变化线和坡度数值标签 |
| 曲线半径图 | 里程-曲线半径图，显示半径数值标签 |
| 车站信息 | 读取车站列表文件，在图表上标记车站位置、名称和里程 |
| 他轨道叠加 | 在同一画面中叠加显示多条其他轨道（如复线、侧线） |
| 数据导出 | 平面图/纵断面图保存为 PostScript 文件；轨道坐标导出为 CSV 文件 |
| 多语言支持 | 日本語 / English / 简体中文 |

### 1.2 坐标计算原理

Kobushi 在以下两类控制点上计算轨道坐标：

1. **地图要素控制点**：地图文件中各要素（Curve、Gradient、Turn、Track 等）被指定的里程位置
2. **等间距控制点**：默认以 25m 间隔在第一车站位置至最后车站+500m 范围内均匀补充的点

在每两个相邻控制点之间，Kobushi 根据曲线要素计算：
- **水平几何**：直线、圆曲线、回旋线（clothoid）过渡曲线 + sin 半波长递减函数
- **垂直几何**：恒定坡度、竖曲线过渡
- **超高（Cant）**：线性或 sin 半波长递减插值

- **操作系统**：Windows / macOS / Linux（支持 tkinter 的图形环境）

```powershell

```

python -m kobushiM "path/to/mapfile.txt"

### 3.1 工具栏

- **打开按钮**：点击打开文件选择对话框，选择 BVE 地图文件（.txt）
- **文件路径输入框**：显示当前打开的地图文件路径，也可直接输入

### 3.2 控制面板 — 辅助信息

位于主窗口右侧，包含 7 个复选框，用于控制在平面图和纵断面图上显示的辅助信息：

| 复选框 | 功能描述 |
|--------|----------|
| **车站坐标** | 在 `Station[*].Put` 设定的位置显示 ● 符号 |
| **车站名称** | 在车站符号旁显示车站名称（依赖「车站坐标」开启） |
| **车站里程** | 在车站符号旁显示里程标（黄色文字） |
| **坡度变化点** | 在纵断面图上用白色竖线标记坡度变化位置 |
| **坡度数值** | 在纵断面图上显示坡度值（‰）（依赖「坡度变化点」开启） |
| **曲线半径** | 在曲线半径图上用垂直文字显示半径值 |
| **纵断面他轨道** | 在纵断面图上叠绘其他轨道的标高线 |

默认状态：除「纵断面他轨道」默认关闭外，其余全部开启。

### 3.3 控制面板 — 图表显示

| 复选框 | 功能描述 |
|--------|----------|
| **坡度图 (Gradient Graph)** | 控制纵断面图区域的显示/隐藏 |
| **曲线图 (Curve Graph)** | 控制曲线半径图区域的显示/隐藏 |

两个图表可以分别独立开启或关闭。完全关闭后，对应的画布区域会被隐藏以最大化剩余图表空间。

### 3.4 底部工具条

- **车站跳转下拉框**：列出地图中所有车站（格式：`stationkey, StationName`），选择一个车站即可将三个画布的视图中心跳转至该车站位置

## 5. 他轨道窗口

他轨道窗口是一个独立的子窗口，用于管理地图中其他轨道（Other Track）的显示设置。

### 5.1 打开方式

加载地图文件后，他轨道窗口会自动弹出在主窗口右侧。如果关闭了该窗口，重新加载地图文件即可再次打开。

### 5.2 界面结构

他轨道窗口使用 **可勾选树形列表（CheckboxTreeview）**，包含以下列：

| 列名 | 说明 |
|------|------|
| **track key** | 轨道标识符（地图文件中 `Track['key']` 定义的 key） |
| **From** | 该轨道显示的起始里程 [m] |
| **To** | 该轨道显示的终止里程 [m] |
| **Color** | 该轨道在图表上的显示颜色 |

### 5.3 操作方法

#### 勾选轨道

- 勾选某个轨道 key 对应的复选框，即可在平面图和纵断面图上显示该轨道
- 勾选/取消勾选最顶部的 **root** 节点，可以一键全选/全不选
- 默认状态：全部不勾选

#### 修改显示区间（From / To）

- 点击某个轨道的 **From** 或 **To** 列的值
- 在弹出的输入框中输入新的里程值
- From 的默认值是轨道要素首次出现的里程
- To 的默认值是轨道要素最后出现的里程

#### 修改显示颜色

- 点击某个轨道的 **Color** 列（显示为 ■■■ 色块）
- 在弹出的系统颜色选择器中选择新颜色
- 颜色会立即在图表上生效

#### 默认颜色

Kobushi 默认使用 10 种颜色轮换分配给不同的他轨道：
`#1f77b4` `#ff7f0e` `#2ca02c` `#d62728` `#9467bd` `#8c564b` `#e377c2` `#7f7f7f` `#bcbd22` `#17becf`

## 6. 画布交互操作

Kobushi 有三个画布：平面图、纵断面图、曲线半径图。每个画布支持以下交互操作：

### 6.1 平面图（Plan View）

| 操作 | 方式 | 说明 |
|------|------|------|
| **平移** | 鼠标左键拖拽 | 拖动画布平移视图 |
| **旋转** | 鼠标右键拖拽 | 以画布中心为轴旋转视图 |
| **缩放** | 鼠标滚轮 | 以水平方向为主缩放（X轴方向缩放） |
| **Y 轴缩放** | `Shift + 鼠标滚轮` 或 `Shift + 右键拖拽` | 在旋转模式下，Shift+滚轮为旋转 |
| **重置视图** | 双击左键 | 自动适应窗口大小显示全部内容 |

平面图的特性：
- **Y 轴向下**（屏幕坐标系）
- 自带**比例尺**（右下角）
- 背景为**80px 间距的像素网格**

### 6.2 纵断面图（Profile View）

| 操作 | 方式 | 说明 |
|------|------|------|
| **平移** | 鼠标左键拖拽 | 拖动画布平移视图 |
| **X 轴缩放** | 鼠标滚轮（默认） | 横向缩放（里程方向） |
| **Y 轴缩放** | `Shift + 鼠标滚轮` | 纵向缩放（标高方向） |
| **整体缩放** | `Ctrl + 鼠标滚轮` | X、Y 轴同步缩放 |
| **重置视图** | 双击左键 | 自动适应窗口大小 |

纵断面图的特性：
- 带有**世界坐标网格**，网格线上标注里程（m）和标高（m）
- 具有独立的 X、Y 缩放比例
- 不支持旋转
- 默认以 X 轴缩放优先

### 6.3 曲线半径图（Curve Radius View）

| 操作 | 方式 | 说明 |
|------|------|------|
| **平移** | 鼠标左键拖拽 | 拖动画布平移视图 |
| **X 轴缩放** | 鼠标滚轮（默认） | 横向缩放（里程方向） |
| **Y 轴缩放** | `Shift + 鼠标滚轮` | 纵向缩放（半径方向） |
| **整体缩放** | `Ctrl + 鼠标滚轮` | X、Y 轴同步缩放 |
| **重置视图** | 双击左键 | 自动适应窗口大小 |

曲线半径图的特性：
- Y 轴中心锁定（Y=0 始终位于画布垂直中心）
- 带有世界坐标网格
- 具有独立的 X、Y 缩放比例

### 6.4 通用快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+O` | 打开地图文件 |
| `Ctrl+S` | 保存图像 |
| `F5` | 重新加载地图文件 |
| `Alt+F4` | 退出程序 |

## 7. 选项设置

### 7.1 坐标控制点（Control Points）

调整 Kobushi 生成轨道坐标时使用的等间距控制点参数：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| **min（最小值）** | 第一车站里程 | 等间距控制点的起始里程 |
| **max（最大值）** | 最后车站里程 + 500m | 等间距控制点的终止里程 |
| **interval（间隔）** | 25m | 控制点之间的间距 |

**操作步骤**：
1. 点击菜单 **Options → 座標制御点...**
2. 在弹出的对话框中输入新的 min、max、interval 值
3. 点击 **OK** 确认，或 **Reset** 恢复默认值
4. 确认后，程序会自动重新加载地图并重新计算轨道坐标

> **注意**：过小的间隔值配合大范围可能消耗大量内存，请谨慎设置。

### 7.2 绘图区间（Plot Limit）

设定「全范围」模式下，平面图和纵断面图可显示的里程上下限：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| **min（最小值）** | 第一车站里程 - 500m | 可显示区间的起始里程 |
| **max（最大值）** | 最后车站里程 + 500m | 可显示区间的终止里程 |

**操作步骤**：
1. 点击菜单 **Options → 描画可能区間...**
2. 在对话框中输入 min、max 值
3. 点击 **OK** 确认，或 **Reset** 恢复默认值

### 7.3 字体设置（Font）

指定平面图、纵断面图、曲线半径图上所有文字使用的字体：

**操作步骤**：
1. 点击菜单 **Options → フォント...**
2. 在下拉列表中选择系统可用字体
3. 点击 **OK** 确认，图表上的文字即时更新

也可以通过命令行参数 `-f` 指定（见[第9章](#9-命令行参数)）。

默认字体：`TkDefaultFont`（系统默认 sans-serif 字体）

## 8. 数据导出

### 8.1 保存图像

将三个画布（平面图、纵断面图、曲线半径图）的当前内容保存为 PostScript 文件：

**操作步骤**：
1. 点击菜单 **File → 画像を保存...** 或按 `Ctrl+S`
2. 在保存对话框中选择保存目录并输入文件名基准（如 `myroute`）
3. 程序将生成三个文件：
   - `myroute_plan.ps` — 平面图
   - `myroute_profile.ps` — 纵断面图
   - `myroute_radius.ps` — 曲线半径图

文件格式选项：
- **PostScript（.ps）** — 默认格式，可转换为 PDF/PNG
- **any format（任意格式）** — 可手动指定其他扩展名

### 8.2 保存轨道数据

将自轨道和他轨道的坐标数据导出为 CSV 文件：

**操作步骤**：
1. 点击菜单 **File → 走行位置情報を保存...**
2. 在对话框中选择**输出目录**
3. 程序在所选目录下生成以下文件：
   - `<目录名>_owntrack.csv` — 自轨道数据
   - `<目录名>_<trackkey>.csv` — 每个他轨道各一个文件

CSV 格式详见[附录](#13-附录csv数据格式)。

## 9. 命令行参数

Kobushi 支持以下命令行参数：

| 参数 | 说明 | 示例 |
|------|------|------|
| `filepath` | 启动时直接打开的地图文件路径 | `python -m kobushi route.txt` |
| `-s STEP, --step STEP` | 轨道坐标计算的等间距控制点间隔 [m]（默认 25） | `-s 10` |

**示例**：

```powershell
# 以 10m 间隔计算轨道坐标
python -m kobushi -s 10 "my_route.txt"

# 指定字体，以 50m 间隔计算
python -m kobushi -f "Arial" -s 50 "my_route.txt"

# 仅启动 GUI，手动选择文件
python -m kobushi
```

## 12. 常见问题与注意事项

### 12.1 地图文件兼容性

以下类型的地图文件**可能加载失败**：

- **BVE Map 1.x 格式**：请使用 [Map Converter](https://bvets.net/jp/download/mapconv.html) 转换为 Map 2.00 格式
- **变量名包含多字节字符**：从 v1.1.7 开始已支持
- **变量赋值了地图要素**：不支持将地图要素（Track、Curve 等）赋值给变量
- **地图要素 key 未用单引号括起**：`Track[hoge]` 应写为 `Track['hoge']`
- **Header 编码声明与实际编码不一致**
- **其他不符合 [Map 2.02+ 语法规范](https://bvets.net/jp/edit/formats/route/map.html) 的写法**

### 12.2 编码检测

Kobushi 读取文件时按以下逻辑检测编码：

1. 读取文件 Header 中指定的编码
2. 如果读取失败：
   - Header 指定 UTF-8 → 尝试 Shift-JIS（CP932）
   - Header 指定 Shift-JIS → 尝试 UTF-8

### 12.3 内存使用

在极长路线 + 极细控制点间隔的场景下，计算数据可能占用大量内存。建议在必要时才调小控制点间隔。

### 12.4 他轨道 Y 坐标

v1.2.2 修复了他轨道 Y 坐标的计算方法。如果你发现他轨道在平面图上的位置不正确，请升级到最新版本。

## 13. 附录：CSV 数据格式

### 13.1 自轨道 CSV（`*_owntrack.csv`）

列名（Header）：

```
distance,x,y,z,direction,radius,gradient,interpolate_func,cant,center,gauge
```

| 列名 | 单位 | 说明 |
|------|------|------|
| `distance` | m | 里程 |
| `x` | m | X 坐标 |
| `y` | m | Y 坐标 |
| `z` | m | Z 坐标（标高） |
| `direction` | radian | X 轴方向的方位角 |
| `radius` | m | 曲线半径（直线 = 0，右转 = 正，左转 = 负） |
| `gradient` | ‰ | 坡度（千分比） |
| `interpolate_func` | — | 缓曲线函数类型：`0` = sin 半波长递减，`1` = 直线递减 |
| `cant` | m | 超高量 |
| `center` | m | 超高旋转中心 |
| `gauge` | m | 轨距 |

### 13.2 他轨道 CSV（`*_<trackkey>.csv`）

列名（Header）：

```
distance,x,y,z,interpolate_func,cant,center,gauge
```

| 列名 | 单位 | 说明 |
|------|------|------|
| `distance` | m | 里程 |
| `x` | m | X 坐标 |
| `y` | m | Y 坐标 |
| `z` | m | Z 坐标（标高） |
| `interpolate_func` | — | 缓曲线函数类型 |
| `cant` | m | 超高量 |
| `center` | m | 超高旋转中心 |
| `gauge` | m | 轨距 |
